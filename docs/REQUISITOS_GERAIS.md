## Requisitos Funcionais

- RF-01. O sistema deve permitir cadastro de usuários.
- RF-02. O sistema deve permitir login dos usuários cadastrados.
- RF-03. O sistema deve permitir Gerenciar o perfil do usuário
- RF-04. O sistema deve permitir Buscar e-books por título, autor ou categoria.
- RF-05. O sistema deve permitir Ler e-books em múltiplos dispositivos com sincronização automática.
- RF-06. O sistema deve permitir Baixar e-books para leitura offline.
- RF-07. O sistema deve permitir Marcar páginas, anotar e destacar trechos.
- RF-08. O sistema deve permitir Gerenciar assinatura (renovação, cancelamento).
- RF-09. O sistema deve permitir Receber pagamentos mensais automaticamente.
- RF-10. O sistema deve permitir Gerenciar catálogo e fazer upload em lote.
- RF-11. O sistema deve permitir Controlar direitos autorais e licenciamento.
- RF-12. O sistema deve permitir Visualizar relatórios consolidados de consumo e receita.
- RF-13. O sistema deve permitir Gerenciar usuários e moderar conteúdo.
- RF-14. O sistema deve permitir Configurar regras de distribuição de receita.
- RF-15. O sistema deve permitr Monitorar saúde da plataforma e gerar relatórios financeiros.

## Requisitos Não-Funcionais

- RNF-01. O sistema deve responder às buscas no catálogo de e-books em até 2 segundos para pelo menos 95% das requisições.
- RNF-02. O sistema deve permitir a abertura de um e-book para leitura em até 3 segundos após a seleção pelo usuário.
- RNF-03. A sincronização do progresso de leitura entre dispositivos deve ocorrer em até 5 segundos após a atualização.
- RNF-04. O sistema deve exigir autenticação para acesso a funcionalidades restritas, como leitura, publicação e administração.
- RNF-05. As credenciais dos usuários devem ser armazenadas de forma segura, utilizando mecanismos de criptografia de senha.
- RNF-06. O sistema deve garantir controle de acesso baseado em perfis (Leitor, Autor, Editora e Administrador).
- RNF-07. As transações financeiras devem ser realizadas por meio de conexão segura, utilizando protocolo HTTPS.
- RNF-08. O sistema deve apresentar disponibilidade mínima de 99% ao mês.
- RNF-09. O sistema deve registrar falhas e erros para posterior análise pelo Administrador.
- RNF-10. O sistema deve preservar o progresso de leitura do usuário em caso de falhas ou interrupções inesperadas.
- RNF-11. O sistema deve ser compatível com navegadores web e dispositivos móveis nas plataformas iOS e Android.
- RNF-12. A interface do sistema deve se adaptar a diferentes tamanhos de tela, mantendo a usabilidade.
- RNF-13. O sistema deve permitir que um novo usuário consiga explorar o catálogo e iniciar a leitura de um e-book sem necessidade de treinamento prévio.
- RNF-14. As funcionalidades principais do sistema devem estar acessíveis em no máximo 3 interações a partir da tela inicial.
